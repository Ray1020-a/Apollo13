# 遊戲設計文件 (GDD)：《阿波羅 13 號：極限斷電》(Apollo 13: Power Triage)

---

## 1. 遊戲概述

- **遊戲類型**：純 UI 互動 / 極限生存資源管理
- **視覺風格**：整體為復古像素風格（Pixel Art），核心儀表板為 1970 年代黑白點陣螢幕（Dot Matrix），帶有陰極射線管（CRT）的掃描線與閃爍殘影效果
- **核心體驗**：拔除複雜的教學與數值面板，透過視覺（畫面結霜、發黑）與聽覺（急促警報、沉重心跳），迫使玩家在「失去自動化系統」的高壓極限下，進行手動分時操作，體驗化解危機的瞬間快感
- **一局時長**：8 分鐘（480 秒）

---

## 2. 歷史與故事背景

- **真實歷史基底**：1970 年 4 月 13 日，任務計時（Mission Elapsed Time）55:55，服務艙（Service Module）的 2 號液態氧槽因電線絕緣層受損引發大爆炸，導致主電力與維生系統癱瘓，太空人被迫退守登月艙（Lunar Module）Aquarius 盲飛求生。事故到濺落歷時 **87 小時**，遊戲將此壓縮為 8 分鐘（壓縮比例：**652.5 倍**）。
- **遊戲機制合理化**：
  - **自動調節器燒毀**：爆炸引發電湧，摧毀負載平衡電腦，玩家必須手動分配僅存電流
  - **冷卻管線破裂**：設備無法自動冷卻，開啟過久就會過熱當機
  - **熱力學反應**：釋放液態氧時，氣體膨脹瞬間帶走艙內熱能（真實物理現象），造成急凍危機

---

## 3. 開場過場動畫

## 4. 勝負條件

- **勝利**：ETA 倒數至 `00:00:00` 且總電力 > 0%
- **失敗（Game Over）**：
  1. **永遠漂流**：總電力（MAIN PWR）歸零
  2. **窒息身亡**：艙內氧氣（O2 TANK）歸零
  3. **中毒身亡**：二氧化碳（CO2 LVL）達到 10000 PPM
  4. **失溫凍死**：艙內溫度（TEMP）降至 0°C
  5. **迷航燒毀**：軌道偏離度（DEV）超過 50%（再入角度過大，大氣層中燒毀）

## 5. 參數系統（全部數值）

### 5-A. 抵達時間（ETA）

| 項目 | 數值 |
|------|------|
| 初始值 | `87:00:00` |
| 倒數速率（導航開啟時） | 正常倒數（652.5 倍壓縮） |
| 導航關閉時 | ETA **暫停**，DEV 每累積 1% 額外增加 0.5 秒 |
| DEV 超過 30% | ETA 獲得永久 +30 秒懲罰（軌道嚴重偏移，無法完美再入） |
| 歸零 | 觸發勝利 |

---

### 5-B. 主電力（MAIN PWR）

| 項目 | 數值 |
|------|------|
| 初始值 | 100% |
| 消耗方式 | 各設備安培數 × 時間（詳見設備表） |
| 換算 | 每安培每秒消耗 **0.025%** 電力 |
| 歸零 | 觸發失敗 |
| 特性 | 只降不升，不可逆 |

**各設備耗電速率**：

| 設備 | 安培（A） | 每秒耗電 |
|------|-----------|----------|
| 暖氣（HEATER） | 3A | 0.075%/秒 |
| CO2 過濾器（CO2 FILTER） | 5A | 0.125%/秒 |
| 導航電腦（NAV COMP） | 4A | 0.1%/秒 |
| O2 釋放閥按住時（O2 RELEASE） | 2A | 0.05%/秒 |
| 過濾器 + 導航同時開（9A） | 9A | 0.225%/秒 → 480 秒消耗 108%，**必須點放** |
| 三個全開（12A） | 12A | 超過紅線，引發跳電 |

---

### 5-C. 艙內氧氣（O2 TANK）

| 項目 | 數值 |
|------|------|
| 初始值 | 100% |
| 自然消耗 | -0.15%/秒（3 名組員呼吸），100% 約在 667 秒（11 分鐘）耗盡 |
| 歸零 | 觸發失敗 |
| 補充方式 | 長按 O2 釋放閥，從液態氧儲備（LIQ O2）補充 |

**液態氧儲備（LIQ O2）子系統**：

| 項目 | 數值 |
|------|------|
| 初始罐數 | 3 罐（畫面以 3 個罐子圖示表示） |
| 釋放速率 | 長按時：O2 TANK +0.8%/秒，TEMP -0.6°C/秒 |
| 每罐容量 | 支撐約 10 秒長按（= +8% O2，-6°C） |
| 耗盡後 | O2 RELEASE 按鈕失效，無法再補充 |
| 真實根據 | 登月艙 Aquarius 自帶下降段與上升段兩套液態氧系統，服務艙爆炸後僅剩此來源 |

---

### 5-D. 二氧化碳濃度（CO2 LVL）

| 項目 | 數值 |
|------|------|
| 初始值 | 400 PPM（接近地球正常值） |
| 自然累積 | +20 PPM/秒（3 名組員呼吸，無過濾器時） |
| 致命值 | 10000 PPM（從 400 PPM 開始，**不開過濾器剛好 480 秒後死亡**，即第 8 分鐘） |
| 視覺閾值 1 | 超過 5000 PPM：畫面四周出現黑暈（約第 3 分 50 秒觸發） |
| 視覺閾值 2 | 超過 8000 PPM：滑鼠游標開始隨機漂移（模擬頭暈） |
| 歸零方式 | 開啟 CO2 過濾器：-40 PPM/秒（淨值 -20 PPM/秒） |
| 真實根據 | 阿波羅 13 號 CO2 濾罐為方形，登月艙孔為圓形，地面用膠帶、塑膠袋、說明書紙管即興拼出轉接裝置「郵箱（Mailbox）」，以無線電指導太空人組裝 |

---

### 5-E. 艙內溫度（TEMP）

| 項目 | 數值 |
|------|------|
| 初始值 | 21°C |
| 自然冷卻 | -0.035°C/秒（**直接由真實數據換算**：87 小時從 21°C 降至 4°C，乘以 652.5 倍壓縮） |
| 不開暖氣的 8 分鐘結果 | 21 - (0.035 × 480) = **4.2°C**，完全符合真實歷史紀錄的 4°C |
| 致命低溫 | 0°C，觸發失敗 |
| 暖氣升溫速率 | +0.1°C/秒（淨值 +0.065°C/秒） |
| O2 釋放副作用 | 長按閥門時 -0.6°C/秒（氣體膨脹帶走熱能） |
| 視覺閾值 1 | 低於 10°C：螢幕邊緣出現霜花 |
| 視覺閾值 2 | 低於 5°C：霜花遮住部分儀表數字（MAIN PWR 或 DEV 被遮） |
| 低溫對導航影響 | 低於 10°C 時，陀螺儀（Gyroscope）因熱脹冷縮影響精度，DEV 漂移範圍從 0~2% 擴大至 **0~3%**（見 5-F） |
| 真實根據 | 阿波羅 13 號真實艙溫降至低於 40°F（約 4°C），組員每人每日僅 6 盎司飲水配給 |

---

### 5-F. 軌道偏離度（DEV）

| 項目 | 數值 |
|------|------|
| 初始值 | 0% |
| 導航開啟時 | 顯示真實數值，每秒 **-1%** 校正 |
| 導航關閉時（盲飛） | 顯示 `ERR`，每秒隨機漂移 **0~2%**（均勻隨機） |
| 低溫加劇漂移 | 溫度低於 10°C 時，盲飛漂移範圍擴大至 **0~3%/秒** |
| 期望值分析 | 正常漂移平均 1%/秒 = 校正速率，短暫盲飛有機率划算；低溫時平均 1.5%/秒 > 校正速率，盲飛開始虧本 |
| 危險閾值 | DEV > 30%：ETA 獲得永久 +30 秒懲罰 |
| 致命閾值 | DEV > 50%：觸發失敗（再入角度過大燒毀） |
| 真實根據 | 阿波羅 13 號關閉指令艙導航後，爆炸殘骸漂浮在飛船周圍，太空人無法從窗戶分辨星星與碎片，IMU（慣性測量單元）重新校準極為困難 |

## 6. 設備系統

### 6-A. 設備一覽

| 設備 | 類型 | 安培 | 功能 | 過熱上限 | 過熱鎖定 |
|------|------|------|------|----------|----------|
| 暖氣（HEATER） | 撥動開關 | 3A | TEMP +0.1°C/秒 | **連續 12 秒** | 5 秒 |
| CO2 過濾器（CO2 FILTER） | 撥動開關 | 5A | CO2 -40 PPM/秒 | **連續 8 秒**（最快過熱） | 5 秒 |
| 導航電腦（NAV COMP） | 撥動開關 | 4A | ETA 倒數 + DEV -1%/秒 | **連續 18 秒** | 5 秒 |
| O2 釋放閥（O2 RELEASE） | 長按按鈕 | 2A（按住時） | O2 +0.8%/秒，TEMP -0.6°C/秒，消耗液態氧罐 | 無過熱限制 | — |

### 6-B. 安培負載表（AMP LOAD）

- **紅線位置**：超過 **10A** 進入危險區
- **安全上限**：同時最多開兩個主要設備（3+4=7A 或 3+5=8A 或 4+5=9A）
- **三個全開**：3+5+4 = **12A**，超過紅線，2 秒後觸發跳電
- **跳電三層懲罰**：
  1. **立即**：所有設備強制斷電，畫面全黑 1 秒，需點擊 RESET 恢復（反應時間即為懲罰）
  2. **永久**：扣除 **3% 總電力**（電壓突波燒損電池容量，真實物理現象）
  3. **機率性**：推超紅線的最後那個設備有 **50% 機率永久降效**（例如過濾器降效後 CO2 清除速率從 40 降至 25 PPM/秒），以警示圖示標記

## 8. 遊戲流程與心流體驗

以「施加極限壓力 → 解消壓力 → 獲得快感」的設計哲學為核心：

1. **平靜的消耗（盲飛省電期）**：玩家關閉導航開始盲飛，ETA 凍結，DEV 在後台悄悄以 0~2% 的隨機速度攀升，玩家心裡盤算：「再撐 5 秒就好……」
2. **危機浮現**：CO2 濃度飆高，畫面開始發黑；溫度下滑，邊緣結霜
3. **連鎖災難（壓力最高點）**：長按 O2 閥門補充氧氣，溫度因此暴跌，霜花蓋住關鍵數字。慌亂中同時打開暖氣與過濾器，安培逼近紅線，設備閃爍過熱警告
4. **軌道校正的致命等待**：開啟導航，面板跳出 `DEV: 14%`，電力狂吃，ETA 重新倒數，但偏離度 `14%... 13%... 12%...` 一格格緩降，玩家只能等
5. **極限微操**：DEV 歸零後立刻關導航省電，點放暖氣與過濾器，在不跳電的邊緣把所有數值拉回安全線
6. **劫後餘生**：結霜退去、視野恢復，短暫喘息——然後立刻面臨新一輪「要不要再盲飛久一點」的心理博弈
